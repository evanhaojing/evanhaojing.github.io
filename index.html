##Django##

###虚拟环境：###
1、安装虚拟环境
pip install virtualenv
2、使用
virtualenv –no-site-packages -p D:python3\python3.6.exe name

a、--no-site-packages: 表示创建的虚拟环境为纯净的环境
b、-p：表示虚拟环境中的python版本

3、pip使用
a、pip list - 查看虚拟环境配置（安装库）
b、pip install xxx - 安装
4、激活虚拟环境
1、Windows：直接执行activate
2、Mac/Linux/Ubuntu:直接执行source activate
5、退出虚拟环境Window/Mac/Linux/Ubuntu
deactivate

###创建项目：###
1、创建
django-admin startproject 项目名称
2、启动
1、启动命令 - python manage.py runserver。默认IP是127.0.0.1 默认端口为8000
2、修改启动端口 - python manage.py runserver 端口
3、修改IP和端口 - python manager.py runserber IP:端口
4、IP参数：如果为0.0.0.0表示任何人都可以通过公网IP访问django项目
5、端口PORT参数：如果端口设置为80，表示该端口不用写

###Django模式：###
MVT/MTV
M: modles.py , 模型层：定义模型和数据库中的表之间的关联关系
V: views.py , 视图层：写业务逻辑的地方
T: templates , 模板：HTML页面

MVT模式是由MVC模式演变出来的

MVC模式是所有框架遵循的模式
MVC
M：模型层
V：模板，HTML页面
C：controller，控制器，写业务逻辑的地方

###数据库准备工作：###
数据库连接（setting.py配置）
USER参数：访问用户
PASSWORD参数：密码
HOST参数：访问数据库的地址
PORT参数：访问数据库的端口
NAME参数：数据库名
‘OPTIONS’:{‘isolation_level’:None}
安装pymysql
1、使用pymysql连接数据库：因为python3没有Mysqlab驱动，无法直接连接mysql

2、在工程目录的 init .py文件中定义
pymysql.install_as_MySQLdb()
迁移
1、第一次迁移
python manage.py migrate
2、除开第一次迁移
a、python manage.py makemigrations 生成迁移文件
b、python manage.py migrate 执行迁移文件
插入管理员账号
python manage.py createsuperuser
